monet — S3: Split Fleet

MULTI-POOL / MULTI-REGION

N worker pools across regions or hardware classes — planner composes across pools via CapabilityIndex

Pool Routing via CapabilityIndex

  • • Worker heartbeats populate CapabilityIndex — planner sees fleet-wide capabilities
  • • RoutingNode carries pool name — workers poll only their pool's queue slice
  • • Planner can compose across pools without knowing where agents run
  • • Pools can be in different regions, on different hardware, behind NAT

Pool "fast" vs "heavy"

  • • fast: Cloud Run — researcher/fast, analyst, qa/fast (quick turnaround)
  • • heavy: Dedicated VPS — researcher/deep, writer/deep, qa/deep, publisher (long tasks)
  • • laptop: ad-hoc personal pool — same agents on your own hardware
  • • Each pool owner controls: model, tools, context injection, OTel exporter

No Inbound Firewall Required

  • • Workers poll Redis outbound — phone/laptop behind NAT works fine
  • • No tunnels, no inbound ports, no exposure of worker machines
  • • Server is the only publicly addressable component
  • • Push pools (Cloud Run, Lambda, Vercel) are declared but dispatcher not yet implemented (roadmap)
