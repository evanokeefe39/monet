monet — S1: Local All-in-One

DEVELOPMENT LAPTOP

monet dev — Aegra + Postgres + Redis in Docker, pool="local" (in-process workers)

Scenario

  • • Single laptop, no remote infra
  • • monet dev starts Aegra + Postgres + Redis via Docker
  • • pool="local" — workers run as background tasks in the server process
  • • MonetClient points to localhost:2026

Key Design Points

  • • Orchestration server (Aegra) is blind to execution details
  • • Only sees: DAG structure + what user leaks into graph state
  • • Agents are opaque capability units — pool="local" is the dev default
  • • ArtifactStore holds pointer-only state (work_brief, RoutingSkeleton)

Separation of Concerns

  • • Server owns: graph orchestration, HITL gates, artifact pointers
  • • Queue owns: task dispatch, polling, pub/sub signals
  • • Workers own: agent runtime, model calls, tool execution
  • • User owns: what goes into graph state (and what doesn't)
