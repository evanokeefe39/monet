monet — S2: Self-Hosted Production

YOUR OWN INFRA

aegra serve + managed Postgres/Redis • monet worker on remote machines • queue decouples orchestration from execution

Scenario

  • • aegra serve + managed Postgres/Redis on your infra
  • • monet worker processes on any machine (VPS, Cloud Run, laptop, phone)
  • • Single MONET_API_KEY shared across server and all workers
  • • Pool name declared in monet.toml — used for routing

Queue Decoupling (core design)

  • • Queue is the ONLY connection between orchestration and execution
  • • Pull model: workers poll Redis, no inbound firewall holes
  • • What goes in the queue: agent_id, command, depends_on
  • • What NEVER goes in: agent code, model calls, tool results, runtime data

Orchestrator Blindness

  • • Server only sees: DAG structure, HITL decisions, artifact pointers
  • • User controls what leaks into graph state via ArtifactStore
  • • Agent runtime, LLM calls, tools, logs — always local to worker
  • • OTel traces from workers can ship to your exporter independently
